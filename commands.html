Top Git Commands You Must Know

Essential commands for everyday Git workflows (command — short description):

  • git init — create a new local repository
  • git clone <url> — copy a remote repository locally
  • git status — show current repo status and staged changes
  • git add <file|.> — stage changes for commit
  • git commit -m "msg" — commit staged changes with a message
  • git commit --amend — amend the previous commit
  • git branch — list, create, or delete branches
  • git checkout <branch> — switch branches (use git switch nowadays)
  • git switch <branch> — modern command to switch or create branches
  • git merge <branch> — merge another branch into the current one
  • git rebase <branch> — reapply commits on top of another base
  • git fetch — fetch remote changes without merging
  • git pull — fetch and merge remote changes
  • git push — upload local commits to the remote
  • git log — view commit history
  • git diff — show changes between commits, branches, or working tree
  • git stash — temporarily save uncommitted changes
  • git reset --hard <commit> — reset working tree and index to a commit (destructive)
  • git remote -v — show remote repositories
  • git tag <name> — create a lightweight or annotated tag
  • git revert <commit> — create a new commit that undoes a previous commit
